超详细搭建属于自己的博客-Git+Node.js+Hexo

前言
我一直有记笔记的习惯，方便自己需要的时候回头看看，主要自己现在还是个菜鸟，需要经常的回顾知识。我在CSDN也有账号，也写过博客，但是界面不友好，就没太大兴趣继续写了，一个偶然的机会，看见别人的博客主页面很好，很清晰。于是乎也想着自己搭建一个自己的博客，创造属于自己的分享天地。Ok，就这样开启了搭建博客之路，在这里我把经验分享给大家，希望大家也多多分享知识，IT行业需要多多交流，不可闭门造车。OK，废话不多说…

1.需要的环境

DaMiss搭建博客的时候本机的系统是WIN10。
Git版本：git version 2.13.0.windows.1
Node.js版本:v6.10.3
Hexo:hexo-cli: 1.0.2
***在这里为什么要强调版本和系统环境，是应为不同版本和不同环境的操作，安装命令的提示都会有所不同,  
因此DaMiss在这里强调一下自己搭建的时候的版本，希望可以帮助更多的小伙伴，少踩坑避免掉坑.***

2.下载并安装msysgit

下载地址：https://git-for-windows.github.io/
安装方法：http://jingyan.baidu.com/album/e52e36154233ef40c70c5153.html?picindex=10
检测是否安装成功：在桌面空白地方点击右键可以看到Git GUI Here，Git Bash Here 即为安装成功，或者在 git bash里面输入命令git -version 如果出现版本号也即为安装成功。

3.下载并安装Node.js

下载地址：https://nodejs.org/en/download/
安装方法：http://jingyan.baidu.com/article/b0b63dbfca599a4a483070a5.html
检测是否安装成功：在cmd里面输入命令node -v 如果出现版本号，即为安装成功。

4.安装hexo

前提：在git和node.js成功安装之后才可以安装Hexo
安装方法：打开cmd命令

npm install hexo-cli -g

等待安装成功…输入下面的命令检测是否安装成功
hexo version

如果出现下面的提示，即为hexo安装成功
hexo-cli: 1.0.2
os: Windows_NT 10.0.10240 win32 x64
http_parser: 2.7.0
node: 6.10.3
v8: 5.1.281.101
uv: 1.9.1
zlib: 1.2.11
ares: 1.10.1-DEV
icu: 58.2
modules: 48
openssl: 1.0.2k

安装成功之后我们就要就开始初始化自己的一个博客，在你本机的某个磁盘下面新建一个存放博客的文件夹，比如E:\myBlog，然后打开cmd 切换到此文件夹下开始输入一下命令

hexo init(初始化自己的博客，安装所有需要的东西)

OK，本地博客初始化工作到此就完成了，接下来我们自己新建一篇博客
hexo new post '你想要的博客名字'
hexo generate（hexo g） //生成静态页面
hexo s //启动hexo服务

完成上述命令之后，我们在浏览器输入网址：http://localhost:4000/ 就可以看到我们本地新建的博客了，hexo默认的主题是landscape，修改主题我会继续在另一篇博客中讲解。

接下来我们看看我们本地初始化的博客的一个目录结构：

myBlog:(刚才init博客的目录)
     -.deploy_git
     -node_modules
     -scaffolds
     -source           //里面存放的我们执行hexo new post 'XXX'产生的博客
     -themes           //主题文件夹，我们修改主题的时候会用到
     -._config.yml.swp
     -.gitignore
     -.npmignore
     -_config.yml      //站点文件，很重要，博客主页面的配置，还有上传到github都要用到
     -db.json
     -package.json

OK，以上所有内容仅限于在我们本地的修改，和使用。但是我们写博客的目的不仅限于自己看，还要分享给大家看，那么问题就来了，我们怎么样才能将我们本地博客搞到一个大家都可以用到的地方呢，DaMiss选择的是gitHub这个免费资源。还不知道gitHub是啥东东的小伙伴，可自行在百度搜索，一搜一大堆呢！OK，闲话少说，我们接着往下看…

首先我们需要在gitHub官网上注册自己的账号和新建自己的仓库。（已经有账号的小伙伴可自行忽略申请账号）

5.申请gitHub账号

官网地址：https://github.com/ 进行注册账号。
新建自己的仓库：注册账号之后，在页面的右上角有个+号，我们点开 new repository ，在这里着重强调一下仓库的名字的规则： yourgitHubname.gitHub.io这种格式的，否则会出现错误，比如DaMiss的gitHub账号是happyfyq ，那我的仓库名字就是happyfyq.github.io.其他默认选择，然后点击 create repository，ok完成!

6.创建密钥

创建密钥的目的是：SSH Keys，看你自己了，可以不配制，不配置的话以后每次对自己的博客有改动提交的时候就要手动输入账号密码，配置了就不需要了。

接下来命令操作需要在git bash 环境下使用，我们右键桌面空白的地方，进入 Git Bash Here ，然后输入一下命令：

ssh-keygen -t rsa -C '你的邮箱'   //这里的邮箱是注册github账号的时候用的邮箱

然后一直按回车，成功之后我们根据提示的保存路径找到一个 id _rsa.pub的文件，复制里面的全部内容。然后登陆的github账号，找到右上角的图标，打开settings,接着找到左面的sshkey接着点击右面的new ssh key，然后把我们刚才复制的东西copy进去，这个密钥的标题随便起。OK密钥创建完成。

接下来我们修改站点目录的配置文件：_config.yml用来上传自己的博客到github上面
主要就是修改下面的内容，打开文件拉倒最底部，修改成我这个样子就好了。repo的地址可以在你的github仓库主页面的找到，我们使用密钥的方式。

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:happyfyq/happyfyq.github.io.git
  branch: master

ok，配置文件搞定，接下来我们就把我们新建一篇博客发布到github上面。
打开cmd：

hexo new post 'XXX'
hexo g
hexo d

Ok ,如果在此过程中没有出现错误，那我们的博客上传到github上面就全部完成，以后每次部署只需要 hexo clean hexo g hexo d 就可以了。

7.出现的问题以及解决办法

1.ERROR Deployer not found: git 或者 ERROR Deployer not found: github

解决：在cmd切换到博客主目录执行：npm install hexo-deployer-git –save

2.执行hexo d 的时候，一直出现提示you have to configure the depoment settings in _config.yml firtst!

解决：在站点配置文件里面配置deploy的时候：repo: git@github.com:happyfyq/happyfyq.github.io.git 这里的repo我一开始写成ssh了，改成repo就好了！
